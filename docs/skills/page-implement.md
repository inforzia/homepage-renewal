## 개요

이 스킬은 화면을 구현하는 방법에 대한 스킬을 지정한 것입니다.

## 목적

- 사용자의 지시에 따라 화면을 구성한다.
- 사용자가 의도한 화면을 의도한 방식대로 구현하는데 초점을 맞춘다.
    - 의도된 화면이란 제공된 원본 디자인 소스(이미지나 웹 페이지)과 디자인이 일치하고, 요청한 구성/기능에 충실한 것을 의미한다.
    - 의도된 방식이란 현재 프로젝트 환경에 적합하며(오류날 여지가 있거나 위험한 코드가 없음), 사용자가 희망하는 컴포넌트 혹은 라이브러리를 이용해 개발하고, 레이아웃이나 구성, 컴포넌트 분리 등을 사용자의 의향, 기존 코드 작성 관습 등에 맞게 따르는 것을 의미한다.

## 요청 분석 방법

- 제공된 것이 스크린샷이라면 해당 스크린샷을 분석하여 필요한 요소를 파악하고 구현 완료 후 스크린샷을 찍어 대조하는 식으로 활용한다
- 제공된 것이 특정 웹사이트라면 CODEX의 `@chrome` 플러그인 도구를 활용해서 분석, 비교하도록 한다. 최종적으로는 viewport 별로 두 사이트 화면을 모두 띄워 사용자가 구현이 정확했는지 확인할 수 있도록 제공한다.
- 단순 지시나 todo 등의 형태로 제공된 경우 지시사항을 명확히 하기 위한 추가 질문을 하거나 관련 파일이나 todo 등을 분석하여 맥락을 이해할 수 있도록 한다.
- 필요하다면 여러가지 분석 방법을 혼용해서 사용한다.
- 분석이 완료되면 해당 내용을 간략히 정리하고 수행해야할 작업이 명시된 todo.md 문서를 생성한다. (파일명은 적절히)

## 작업 방식

지시를 받으면 현재 수행하는 작업에 따라 "역할"을 바꿔가며 작업을 수행합니다. 역할의 종류에는 다음이 있습니다.

- 관리자: 사용자에게 직접 지시를 받고 작업 목록(todo)을 작성하며, 지시사항이 모호하거나 추가적으로 요구되는 정보가 있으면 사용자에게 제안/요청을 하여 지시사항을 명확히 합니다. 이후 작업이 진행될 때마다 경과를 관리하며, 작업 중 추가 소요 작업이 확인되면 todo 목록을 갱신하고, 작업 진행이 불가한 문제가 발생하면 사용자에게 알리고 지시를 기다립니다. 모든 작업이 끝나면 작업 내역을 요약하여 보고하고 완료 여부를 확인받습니다.
- 분석가: `관리자`가 작성한 TODO 및 참고자료등을 분석하여 보다 상세한 TODO 및 참고자료를 확보합니다.
- 작업자: 지시사항에 따라 실제 화면 개발 및 특정 작업을 진행합니다.
- 테스터: 작업된 내용이 의도된대로 제대로 구현되었는지, 오류는 없는지 확인합니다. 정적 검사 및 tsc, lint 등을 중점적으로 활용하되, 해당 지시 내용이 특정 함수의 호출 등 테스트 자동화가 가능한 경우 작업에 앞서 테스트케이스를 만들고 이를 활용할 수도 있습니다.
- 리펙터: 구현된 결과물을 보고 불필요하게 반복되는 코드나 명확히 개선 가능한 부분들을 찾아 적용합니다. 만약 개선의 여지는 보이는데 그것이 기존의 구조를 크게 변경하게 되는 경우 바로 수정하지 않고 보고한 뒤 지시를 기다립니다.

작업은 크게 다음 분류로 나뉜다.

- 주요 작업: 해당 지시를 완료하기 위해 필요한 사항이다. 개략적인 형태로 표현되며 필요시 해당 목록을 바탕으로 상세 작업 목록을 작성한다.
- 상세 작업: 특정 주요 작업을 완료하기 위해 필요한 세부작업들을 의미한다.
- 검토 작업: 작업이 완료된 후 제대로 완료된 것인지 검토하는 작업을 의미한다. 어떤 항목을 어떻게 검토할 것인지 등이 명시된다.

검토 후 완료되었다고 판단되는 작업은 check 표시한다.
